Important note:

From Ghost 0.5.8 there are some changes regards tags image covers.

Now there is a settings panel for post and tags. You can activate it by clicking on cog icon near to 'Publish/Save button' while editing post. You can upload here a post cover image. It will be displayed on index page and post page. You can also go and set up settings for tags.

We resigned from using a hack for post cover images consisting of placing images on top of the post content. From now on it will not work for index page, you should use post settings panel and upload image cover.

Let's see what we've got here:

A) Ghost theme instalation - top

Just copy 'artee' folder to your installed Ghost 'themes' folder and then restart your application. Log into admin area and go to the 'Settings' page. Choose 'artee' from 'Theme' selector. That's it!

B) Artee theme file structure. - top

I use a standard folder structure.

There is an assets folder, index.hbs, post.hbs, page.hbs, error.hbs and default.hbs

There is a partials folder with disqus.hbs, head.hbs, sticky-header.hbs, search.hbs, menu.hbs and sort.hbs files. Here you can configure your disqus comments, post filter blocks and links and also menu links.

There are .scss and .css files in the assets/css folder (you can use the scss files with your favourite Sass workflow. I use the standard Sass command line utility 'scss --watch css:css')

There are 'vendor' folders with all third party plugins in the assets/css and the assets/js folders.

C) Zurb Foundation and Sass - top

You can use Sass compiling. Use assets/css/post.scss and assets/css/style.scss files. They compile to style.css and post.css

You can also use Zurb Foundation (all is configured, Artee theme is based on Zurb Foundation framework) Grid and other Foundation elements, but you must write html in your Ghost markdown editor. All rich documentation of Zurb Foundation framework is here: http://foundation.zurb.com/docs/

In Ghost markdown editor, it’s a good idea to write all your html code on one line without any spaces or breaklines.

D) Working without Sass - top

If you don't want to work with Sass just edit assets/css/style.css and assets/css/post.css files.

How can I do this...

A) How can I add a Youtube or Vimeo videos in my posts? - top

Simply embed code in your Ghost markdown editor. Put it in <div class="flex-video">...</div> See: http://artee.pxt.be/embeding-video/

B) How can I use Zurb Foundation html elements? - top

You can use a couple of Zurb Foundation HTML elements to write in the Ghost markdown editor. Here are some links to Zurb Foundation docs. Remember to remove all white spaces and breaklines from your html blocks when you’re writing in Ghost markdown editor.

Check Zurb Foundation documentation: http://foundation.zurb.com/docs/

Also see this example: http://artee.pxt.be/images-in-grid/

C) How can I use the Mono Social and Font Awesome font icons? - top

Artee theme uses Mono Social Icons: http://drinchev.github.io/monosocialiconsfont/ It is better to use 'Symbol with unicode' for IE compatibility. More in the Mono Social Icons documentation. The docs are really clear and usage is very simple.

Artee theme uses Font Awesome: http://fontawesome.io/ The docs are really clear so you can use them without any configuration.

D) How can I use a gallery carousel and image popup? - top

Artee theme uses Owl Carousel : http://owlgraphic.com/owlcarousel/

General usage in Artee theme:

<div class="artee-gallery">
 <div class="item">
  <a href="/content/images/2014/Jan/1.jpg" class="artee-image-popup">
   <img src="/content/images/2014/Jan/1.jpg" alt="">
  </a>
 </div>
 ...
</div>
 example: http://artee.pxt.be/image-gallery/

If you want Lightbox on image click, simply add 'artee-image-popup' in your <a href=""> link like it is shown.

Artee uses Magnific Popup here: http://dimsemenov.com/plugins/magnific-popup/

You can use Magnific Popup anywhere you want, but remember that in Ghost you still must write html in markdown editor. If you want to place Magnific Popup in (for example) any link you can write this html code:
<a href="path/to/image.jpg" class="artee-image-popup">Some link to image popup</a>

In Ghost markdown editor, it is a good idea to write all your html code on one line without any spaces or breaklines.

E) How can I use Google Maps? - top

Artee theme uses gmaps.js : http://hpneo.github.io/gmaps/

General usage in Artee theme:

<div id="map1" class="gmaps flex-video" data-lat="50.77588" data-lng="15.75560" data-zoom="12"></div>
example: http://artee.pxt.be/contact/

This is the default usage in Artee theme but gmaps.js is very complex and you can try hacking it. Here we use only one div with 'gmaps' and 'flex-video' classes + 3 properties lat, lng and zoom. You can find latitude and longitude params (for example) here: http://universimmedia.pagesperso-orange.fr/geo/loc.htm

Remember the ids of maps must be unique.

You can also change the maps theme. There are a couple of examples in assets/js/vendor/gmaps-styles.js. You can find more at http://snazzymaps.com/ or you can add your own.

If you want to change the maps theme go to assets/js/vendor/artee.js. Find the name of the default theme (one variable) and replace it with one from gmaps-styles.js

In Ghost markdown editor, it is a good idea to write all your html code on one line without any spaces or breaklines.

F) How can I configure the Disqus comments? - top

Artee theme can be fast and simply integrated with the Disqus comments engine.

If you have an account at : http://disqus.com/ create a forum for your Ghost blog open partials/disqus.hbs file and replace Artee theme variable with yours and that’s it.

If you don't want to use Disqus at all, simply open post.hbs and remove the {{> disqus }} part.

G) How can I add some static pages and a menu block? - top

From version 0.4 Ghost allows you to add pages. You can do this by marking a new post as a static page. (cog icon next to the publication/update button)

After that, you can go to your 'Navigation' panel in 'Settings' section. There you can enter links to previously created static pages or any other proper links.

You don’t need to use the menu. If you’d like, just remove all navigation links. More about navigation feature you will find in official Ghost documentation and here: https://blog.ghost.org/navigation/

H) How can I feature a post? - top

This is simple. Click on the star icon for the selected post on the post list view.

Featured posts are marked with a star on the bottom right corner of a single post block. (index view)

I) How can I use the images 'visual diff' plugin? - top

In Artee Theme we use the Twenty Twenty plugin by Zurb: http://zurb.com/playground/twentytwenty It is really awesome. You can use it if you need to highlight the differences between two images. This is some kind of a visual diff tool.

Usage is very simple. Just place your two images in the div container with 'twentytwenty-container' class:

<div class="twentytwenty-container">
  <img src="/content/images/2014/Mar/1-2.jpg" alt="">
  <img src="/content/images/2014/Mar/1c-1.jpg" alt="">
</div>

example: http://artee.pxt.be/twenty-twenty/

J) How can I use post filters on the index page? - top

[New] You can relay on auto populated tags list in filters (all uniq tag names from all visible posts). And you don't need to do anything else. But if you want to choose only few tags and do filtering you should read info below..

Index page post filters are configurable by editing the partials/sort.hbs file. You can add, remove and rename options in a tag filter. Simply edit the option which looks like this: (...)
<li><a href="#" data-option="people">People</a></li>
(...) Change the data-option attribute and change the anchor name. 'data-option' is just a tag used in posts, so if the post has one it will be filtered. Of course you can add as many tags as you want but filter by only some of them. It all depends on the configuration of the sort.hbs file.

So, if you want to add filter options like, for example, 'Photos', 'Drawings', 'Digital' add:

(...)
<li><a href="#" data-option="photos">Photos</a></li>
<li><a href="#" data-option="drawings">Drawings</a></li>
<li><a href="#" data-option="digital">Digital</a></li>
(...)

..and then when you are creating your new post, use the tags field in your admin panel and type 'drawings' or 'photos' or 'digital'.

If you want, you can also remove filters. Just find {{> sort}} in the index.hbs file and remove it. That’s it.

Troubleshooting

A) Error 502 after theme upload - top

This is usually due to owner errors with the files. After copying the theme you need to make sure you set it to be owned by the same user. This should be the same user you are running Ghost as.

If you're using the DigitalOcean Ghost droplet, the command to do this would be
cd /var/www/
followed by
chown -R ghost:ghost ghost/*
